take(1) is useful because it unsubscribes automatically after the first result. It's identical to first().
You can see the state of the subscription in the demo. take(1) unsubscribes even when the observable reports an error (such as an 404 response),
Caveat:take(1) does not subscribe when the server doesn't respond. So it's a potential memory leak.
